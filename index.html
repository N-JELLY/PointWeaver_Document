Point cloud node editor

巨大点群を、ノードで組み替えて確認する。

PointWeaverは、Rustで実装されたWindows向けスタンドアロンの点群処理MVPです。 LAS/LAZ/PLY/PCDを読み込み、3Dビューで確認しながら、式ノードをワイヤー接続して処理を構築します。

概要

GUI必須

編集と確認を同じ画面で

左カラムはエクスプレッションエディタ、右上は3Dプレビュー、右下はノードエディタです。 スプレッドシートはメニューから別ウィンドウとして開きます。

Node

ワイヤー接続で処理順を決定

Source、Expression、Writerを接続します。ワイヤーのつなぎ替えにより、 処理順とプレビュー対象を視覚的に変更できます。

Chunk

巨大点群を前提に分離

LAS/LAZのエクスポートはチャンク単位で処理します。 表示はサンプル/LOD寄りのプレビュー点群を使い、全点描画を避けます。

クイックスタート

開発環境にはRust stableが必要です。リポジトリを取得したら、releaseビルドで実行します。 

git clone https://github.com/N-JELLY/PointWeaver.git
cd PointWeaver
cargo run --release
Windowsで再ビルド時に pointweaver.exe の削除に失敗する場合は、 既に起動しているPointWeaverプロセスを終了してください。

ノードグラフ

処理順

MVPでは1本の直列パイプラインを前提にしています。 SourceからWriterまでの接続を辿り、途中のExpressionノードを順番に実行します。

接続操作

ノード右側の出力ポートをクリックして接続開始、次ノード左側の入力ポートをクリックして接続します。 入力ポートの右クリックで既存接続を外せます。

基本パイプライン

Point Source -> Snap XYZ -> Height To Red -> Writer

式言語

式は実行前に一度パース/検証され、1点ごとに文字列評価されません。 属性はSoA形式のチャンクに対して読み書きされます。

例

X = round(X)
Y = round(Y)
Z = round(Z)
R = remap(Z, minZ, maxZ, 0, 255)
種類利用可能な名前
属性X Y Z R G B intensity classification
統計値minX maxX minY maxY minZ maxZ
演算+ - * / % < <= > >= == !=
関数if round floor ceil clamp remap min max abs sqrt sin cos
remap は入力範囲外も外挿します。勝手なclampは行わないため、 制限したい場合は式側で clamp(...) を明示します。

プレビュー

Final Output

全Expressionノードを通した最終結果を表示します。通常の確認モードです。

Selected Stage

ノードエディター右上のボタンで切り替えます。 選択中の式ノードまで処理した途中段階を表示できます。

プレビュー点数、描画上限、点サイズは3Dプレビュー右下の折りたたみパネルから調整できます。 Z方向や原点はギズモで確認できます。

処理データと表示データ

本体処理はチャンク単位、表示はプレビュー用サンプル点群として扱います。 これにより、数千万から数億点規模のファイルでもGUIスレッドを止めずに扱う設計へ拡張しやすくしています。

用途データ方針
処理PointChunkSoA形式。属性ごとに配列を持ち、チャンク単位で並列処理。
表示PreviewCloudサンプル/LOD用。描画上限を設け、全点を一度に描かない。
統計Stats / Bounds式ノードごとに必要な範囲値を渡す。

アーキテクチャ

UI

eframe/egui を使い、メニュー、エディタ、ノードキャンバス、スプレッドシートを構成します。

3Dビュー

現在はegui PainterベースのMVP表示です。将来の本格表示はwgpu点描画パイプラインに分離します。

Pipeline

GraphDocumentから処理順を決定し、CompiledPipelineが式Programをチャンクへ適用します。

GUI Thread
  -> GraphDocument
  -> CompiledPipeline
  -> SourceCloud / PreviewCloud
  -> Export Worker
  -> PointCloud Writer

対応形式

形式読み込み書き出し備考
LAS対応対応チャンク処理の主経路。
LAZ対応対応las crateのLAZ機能を利用。
PLYASCII対応ASCII対応MVP範囲。巨大ファイル向けチャンクReaderは今後。
PCDASCII対応ASCII対応MVP範囲。binary/compressedは今後。
COPC限定的未対応LAZとして順次読める場合のみ。タイル/LODアクセスは今後。

トラブルシュート

ビルド時にアクセス拒否される

Windowsでは実行中の pointweaver.exe があると、Cargoがrelease exeを上書きできません。 タスクマネージャーかPowerShellで終了します。 

Get-Process pointweaver -ErrorAction SilentlyContinue | Stop-Process

黒い点が出る

色属性の範囲、式の出力、デモデータの色生成を確認してください。 MVPではR/G/Bはプレビュー上で8bit表示されます。

プレビューが少ない/重い

Preview Controlsからサンプル数、Draw limit、Point sizeを調整してください。 巨大点群では処理データと表示データを分けて考えるのが基本です。

ロードマップ

  • PLY/PCD binary対応とチャンクReader/Writer。
  • COPCのタイル/LOD読み込み、オクツリー/タイルキャッシュ。
  • wgpuによる本格点群レンダラ、点サイズ/属性色/選択表示のGPU化。
  • 式言語ASTからWGSL/SPIR-V相当への変換。
  • GPU Computeバックエンド、CPU/GPU実行バックエンド切替。
  • プラグインAPI、Python連携、カスタムノードの外部拡張。
  • 複数入力ノード、分岐/合流、キャッシュ制御、ジョブキュー。
